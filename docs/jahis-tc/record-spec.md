このドキュメントは、JAHIS 電子版お薬手帳データフォーマット仕様書 Ver.2.6 の実装観点の要点を整理するための台帳です。

目的:

- 仕様上の構成を実装者向けに要約する
- バージョン差分と実装差分を切り分ける
- `docs/jahis-tc/spec-coverage.md` と相互参照できる一次メモを残す

関連ドキュメント:

- 実装対応状況: `docs/jahis-tc/spec-coverage.md`
- コード表ノート: `docs/jahis-tc/jahis-tc-codes.md`
- 公開API: `src/index.ts`
- TC パーサ本体: `src/jahis-tc/parse-jahis-tc.ts`

## バージョン情報

- 対象仕様書: JAHIS 電子版お薬手帳データフォーマット仕様書 Ver.2.6
- 技術文書番号: JAHIS 技術文書 24-104
- 仕様上のヘッダ: `JAHISTC08`
- 出力区分:
  - `1`: 医療機関・薬局から患者等に情報を提供する場合
  - `2`: 患者等から医療機関・薬局に情報を提供する場合

実装メモ:

- 現行実装は `JAHISTC08` 固定ではなく、`JAHISTC` + 2桁数字を許容する。
- `outputCategory` は正規化結果の `normalized.outputCategory` に保持される。

## ファイル形式

- 形式: タグ形式の MS-DOS テキストファイル
- 文字種: ASCII または Shift-JIS データ
- 区切り: 半角カンマ `,`
- レコード終端: `CR LF`
- EOF は情報提供形態によって省略可

実装メモ:

- パーサは改行コードを正規化してから処理する。
- 各行は共通の `parseRecord` で分割され、TC 専用デコーダへ渡される。

## 情報グループ

仕様上の主な情報グループは次のとおりです。

- バージョン情報
- 患者基本情報
- 要指導医薬品・一般用医薬品服用情報
- お薬手帳メモ情報
- 調剤情報
- 処方情報
- 薬品情報
- 用法情報
- 処方服用注意情報
- 服用注意情報
- 医療機関等提供情報
- 残薬確認情報
- 備考情報
- 患者等記入情報
- かかりつけ薬剤師情報
- 制御情報

実装メモ:

- 正規化モデルでは、調剤情報は `dispensings[]` 配下にまとまる。
- 調剤ブロックの中に `prescriptions[]` があり、その下に `rps[]` がある。
- `2`、`3`、`4`、`701`、`911` はルート直下の情報として保持される。

## レコード出力順

仕様書では、概ね次の順序で出力する。

1. バージョンレコード
2. 患者系レコード `1`、`2`
3. OTC/メモ系レコード `3`、`31`、`4`
4. 調剤ブロック単位のレコード `5` 以降
5. かかりつけ薬剤師 `701`
6. 分割制御 `911`

調剤ブロック内では、概ね次の流れを前提とする。

1. `5` 調剤等年月日
2. `11` 調剤－医療機関等
3. `15` 調剤－医師・薬剤師
4. `51` 処方－医療機関
5. `55` 処方－医師
6. `201` / `281` / `291`
7. `301` / `311` / `391`
8. `401` / `411` / `421` / `501` / `601`

実装メモ:

- 実装は `5` を調剤ブロック開始とみなし、これ以前に調剤配下レコードが来ると順序エラーを報告する。
- `201` と `301` の対応は同一 RP 番号で検証する。

## 主要レコードの項目メモ

### ヘッダ

- 形式: `JAHISTC08,出力区分`
- 実装上は `JAHISTC` + 2桁数字なら受理対象
- `outputCategory` は `1` または `2`

### `1` 患者情報

- 必須中心項目: 氏名、性別、生年月日
- 任意項目: 郵便番号、住所、電話番号、緊急連絡先、血液型、体重、氏名カナ
- 実装では体重書式、電話、郵便番号の簡易検証を行う

### `3` 要指導医薬品・一般用医薬品服用

- 実装保持項目: 薬品名称、服用開始日、服用終了日、作成者、レコード通番、JAN コード
- `31` と対になる通番で関連付ける

### `31` 要指導医薬品・一般用医薬品成分

- 実装保持項目: 要指導医薬品・一般用医薬品レコード通番、成分名、コード種別、成分コード、作成者
- 正規化では対応する `3` の `ingredients` 配下へ紐付ける
- コード種別が `1` 以外なら成分コードを必須扱い

### `11` 調剤－医療機関等

- 出力区分 `1` では都道府県、点数表、コードを必須扱い
- 都道府県は `01` から `47`
- 点数表は `1/3/4`

### `51` 処方－医療機関

- 出力区分 `1` で薬品系レコードを出す場合の中核レコード
- 点数表は `1/3`

### `201` 薬品

- 実装保持項目: RP番号、薬品名称、用量、単位名、薬品コード種別、薬品コード、作成者
- 条件付き必須: 薬品コード種別が `1` 以外なら薬品コード必須
- 未保持項目: 一般名、一般名コード種別、一般名コード
- Ver.2.6 では選定療養に関する補足注記が追加されている

### `301` 用法

- 実装保持項目: RP番号、用法名称、調剤数量、調剤単位、剤形コード、用法コード種別、用法コード、作成者
- 出力区分 `1` かつ剤形 `9/10` 以外では `用法名称` を必須扱い
- 用法コード種別が `1` 以外なら用法コード必須

### `411` 医療機関等提供情報

- 実装保持項目: 内容、提供情報種別、作成者
- 種別は `30/31/99`

### `421` 残薬確認

- 実装保持項目: 残薬内容、作成者
- 正規化では `dispensings[].remainingMedicineConfirmations` に保持する

### `701` かかりつけ薬剤師

- 氏名、勤務先薬局名称、連絡先、担当開始日、担当終了日、作成者を保持

### `911` 分割制御

- `dataId`: 14桁数字
- `totalParts`: 分割数
- `partNumber`: 連番
- 実装では `partNumber <= totalParts` を検証
