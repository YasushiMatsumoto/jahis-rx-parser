### 共通インターフェース仕様 (v1.10)

#### 3.1 バージョン情報

- 本規約 Ver.1.10 の先頭行は `JAHIS10` を出力する。
- バージョン情報はファイル先頭行に明示する。

#### 3.2.1 ファイル形式

- 形式: タグ形式の MS-DOS テキストファイル
- 文字種: ASCII または Shift-JIS データ
- 区切り: 半角カンマ `,`
- レコード終端: `CR LF`
- ファイル終端: `EOF`

#### 3.2.2 データ型

| Type | Meaning                    |
| ---- | -------------------------- |
| `9`  | 数値                       |
| `X`  | 英数字、ピリオド、ハイフン |
| `N`  | 文字列（全角半角混在可）   |

#### 3.2.3 ASCIIコード

| Token | ASCII |
| ----- | ----- |
| `,`   | `2Ch` |
| `CR`  | `0Dh` |
| `LF`  | `0Ah` |
| `EOF` | `1Ah` |

#### 3.2.4 注意事項

- 調剤指示の出力先:
  - 薬品に対する情報: `281`（RP薬品補足情報）
  - 剤に対する情報: `181`（RP用法補足情報）
  - 処方箋全体/薬品・剤を特定しない情報: `81`（備考情報）
- 使用制限文字:
  - レセプト電算処理仕様に準拠
  - 半角カンマは区切り文字以外で使用しない
  - 薬品名称等の半角カンマは全角カンマへ置換
  - 全角 `■ (S-JIS 81A1)` は外字置換文字として扱う
- 外字:
  - 外字は使用しない
  - 置換できない場合は `■ (S-JIS 81A1)` に変換して記録
- 引用符:
  - ダブル/シングルコーテーションで囲まない
- レコード省略:
  - 先頭（レコードNo.）から終端（CRLF）まで何も出力しない
- 項目省略:
  - 空文字で表現する
  - 半角/全角スペースは出力しない
- 項目内スペース:
  - 項目先頭/末尾の半角/全角スペースは出力しない

#### 3.2.5 レコード出力順

- レコードは以下の順序で出力する。
  - バージョン情報
  - `1` ～ `82`（医療機関/医師情報グループ）
  - `101` ～ `181`（RP剤情報グループ）
  - `201` ～ `281`（RP内薬品情報グループ）
- `101` ～ `181` と `201` ～ `281` のグループは、RP構成に応じて繰り返される。
- 各グループ内は必ずレコードNo.昇順で出力する。

#### 3.2.7 各種レコードのレイアウト

必須欄記号の凡例:

- `◎`: 必須（当該項目を出力する）。
- `●`: 代替必須（同一レコード内で `●` が付いた項目群のうち、少なくとも1項目を出力する）。
- `-`: 任意（未出力可）。

解釈メモ:

- 70歳以上（高齢受給者）8割は `14,1` または `14,4`、7割は `14,2`。

- 平成20年4月改正時の想定により `4` が存在するが、最終記載要領では「高一」で統一。仕様注記に従い `4` は `1`（高齢者一般）と同義として扱う。
- `5`（高齢者8割・後期高齢者）は令和4年10月診療分から適用。
